Use case run_promer
Description Generate promer data for a comparative database
Actors user,developer
Assumptions/Pre-conditions The BSML_repository and FASTA_repository for the project must be generated via generate_repository. The repository must be in a "valid" state as reported by repository_status.
Post-condition BSML encodings of the promer results are stored in the BSML_repository
Steps

Interactions between actors and system that are necessary to achieve goal

  1. Check repository status via repository_status
  2. Execute run_promer -d [project_name]
  3. Execute workflow monitor on instance filename reported by Step 2.
  4. REPEAT
    1. Monitor workflow status
      UNTIL workflow completed
  5. Execute repository_status
Example invocations /usr/local/devel/ANNOTATION/shared/bin/run_promer -d tryp
Variations (optional) Any variations in the steps of a use case
Non-Functional (optional) The project repository must have sufficient space available to store results.
The user must have permission to write to the project area.
