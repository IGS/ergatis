1. Introduction

  • 1.1 Purpose of this document
    This documentsThis document provides the specification for the software utility set_runtime_vars.

  • 1.2 Scope of this document
    This document should provide the complete specification needed to engineer the set_runtime_vars utility. This document should be updated as needed to reflect the implementation of the software.

     

  • 1.3 Overview
    set_runtime_vars is a utility for replacing placeholder text strings (keys) with values as specified in a simple configuration file.

  • 1.4 Business Context
    This utility is a simple, resuawble component for replacing placeholder keys in text files. This is a common task and a simple utility is needed to handle this task uniformly.

2. General Description

  • 2.1 Product Functions
    A configuration file of key/value pairs is read. The keys are searched against a text file and the key string is replaced with the value from the configuration file. This utility performs a simple search/replace on a text file.

  • 2.2 Similar System Information
    sed is a common UNIX utility that can perform search/replace via regular expressions. sed may be used in the implementation of set_runtime_vars.

  • 2.3 User Characteristics
    This utility should be usable by anyone familiar with the UNIX environment

  • 2.4 User Problem Statement
    There is no standard way to set configuration options for the workflow templates/ini files that are part of cram.

This section lists the functional requirements in ranked order. Functional requirements describes the possible effects of a software system, in other words, what the system must accomplish. Other kinds of requirements (such as interface requirements, performance requirements, or reliability requirements) describe how the system accomplishes its functional requirements. Each functional requirement should be specified in a format similar to the following:

  1. Read configuration file

    1. Description
      Read a simple configuration file specifying key/value pairs

    2. Criticality
      Essential

    3. Technical issues
      The configuration file should follow a simple format. Valid key strings can be any non-white space character. Replacement values can be any non-space character. Key strings should be easily identifiable and fenceposted with delimeters, such as ';', to avoid erroneous matches. 
      	  ;KEY1;=VALUE1
	  ;KEY2;=VALUE2

    4. Cost and schedule

    5. Risks

    6. Dependencies with other requirements

  2. Replace key strings in input file

    1. Description
      Read an input file and replace key strings with replacement values. The key strings and replacement values are input from the configuration file. The input file can be any text file.

    2. Criticality
      Essential

    3. Technical issues
      Given this configuration file: 
      	  ;KEY1;=VALUE1
	  ;KEY2;=VALUE2
      This utility will transform the following document 
      	  <tag1 attribute=';KEY1;'>;KEY2;</tag1>
      and output 
      	 <tag1 attribute='VALUE1'>VALUE2</tag1>

    5. Risks
