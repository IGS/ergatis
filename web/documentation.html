documentation

I'm just throwing some notes here to get an idea of the sort of documents I want to be publicly available. Documentation is currently either being written or decoupled from internal TIGR specifications (much like the rest of the software.)

overview
A description of the overall usage and goals of ergatis will go here. This should include comparisons with other data management packages.
component documentation
A 'component' in ergatis is a tool, script or series of comments that performs a distinct computational task. Examples include format conversion, sequence alignment (BLAST, AAT), or gene prediction programs (augustus, glimmerHMM).
component creation tutorial
Ergatis currently contains 93 components and more are added every month. If there is a specific analysis tool you want to use in the system that doesn't exist as a component you can request it's support or write the component yourself. A tutorial on how this is done will appear here.
directory structure convention
Ergatis creates a lot of files and folders when used, including all output result files. This documentation should describe the standard directory structures employeed.
pipeline creation tutorial
This is where stuff actually gets done. You may have 10,000 sequences you want to BLAST across a compute grid, or a few gene finders to run locally. This is where you learn how to use the interface to build and run your own custom analysis pipeline.
